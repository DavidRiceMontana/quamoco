Description "This method acquires a JSR-166 (java.util.concurrent) lock,
but does not release it on all paths out of the method. In general, the correct idiom
for using a JSR-166 lock is:


Lock l = ...;
l.lock();
try {
// do something
} finally {
l.unlock();
}" [1]
