Title
Description "This rule checks if some methods are decorated with [MethodImpl(MethodImplOptions.Synchronized)]. The runtime synchronize those methods automatically using a lock(this) for instance methods or a lock(typeof(X)) for static methods. This can cause deadlocks if misused inside your own code or if the method is visible outside the assembly since anyone else, outside your control, could be using a lock on an instance or the type and cause a deadlock. The best locking is to create your own, private, instance System.Object and lock on it.
