Description "The code synchronizes on interned String.

private static String LOCK = "LOCK";
...
synchronized(LOCK) { ...}
...


Constant Strings are interned and shared across all other classes loaded by the JVM. Thus, this could
is locking on something that other code might also be locking. This could result in very strange and hard to diagnose
blocking and deadlock behavior. See http://www.javalobby.org/java/forums/t96352.html and http://jira.codehaus.org/browse/JETTY-352." [1]
