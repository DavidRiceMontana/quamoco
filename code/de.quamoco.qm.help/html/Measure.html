Measure

A measure defines how a specific entity is measured. For the concrete measurement, it defines either an instrument, which is bound to a tool or manual measurement, or an aggregation of other measures.

General Rules

  1. Use measures to quantify factors.
  2. Use measures to model checklist items for manual reviews.
  3. For normalizing purposes at an evaluation use normalization measures.

Attributes

Qualified Name

  1. The qualified name is generated automatically based on the name and the characterizing entity.

Name

  1. Choose a short, descriptive name for the measure.
  2. If there is no specific reason to do otherwise, only captialise the first word in the name.

Characterizes

  1. Specify the entity that is characterized by the measure.

Title

  1. A title can be an alternative, more readable identifier for the measure.
  2. Providing a title is optional.
  3. If there is no specific reason to do otherwise, only captialise the first word in the title.

Description

  1. Describe what the measure is intended to measure.
  2. What does it actually measure?

Type

  1. Set one of NONE, FINDINGS, NUMBER.
  2. In many cases, FINDINGS is an appropriate type. Measures that represent rules that are checked on a product can identify locations in the product where the rule is violated. Such a location is called finding. Most of static code analysis tools, for example, return findings.
  3. Measures can return an arbitrary real numerical value. The data type to represent them is NUMBER. This number can be an absolute value or a ratio of some kind, it is mostly appropriate for numerical measures.

Refined by / Refines

  1. A measure can be consists of other measures. In this case, a measure is refined by other measures.
